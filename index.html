예약 취소율이 A 26.84% vs B 26.83%로 거의 같음(차이 0.01%p). 여기에 B 노출자는 97.7%(672/688)가 결제를 완료. 즉 차종UP 바텀시트를 띄워도 기존 예약 전환을 떨어뜨리지 않음
버킷팅 공식은 서버 코드(socar-common Testing.kt)로 확인: farmhash(member_id + 'UPSELLING_CAR_UPGRADE') 기준 50 미만이 노출군. 노출자 688명 전원 노출군에 속함을 BQ로 검증
검증용 실제 쿼리 BigQuery
1. 노출, 옵션 선택, CTA 퍼널 (웹 이벤트)
-- 차종UP은 form_data에 co(원본 차종), cu(제안 차종) 키가 있어 cu 존재로 식별 SELECT CASE WHEN event_name='view' THEN '1_노출' WHEN component_name='car_upgrade_option' THEN '2_옵션선택' WHEN component_name='cta_button' THEN '3_CTA' END AS step, COUNT(DISTINCT member_id) AS uu FROM `socar-data.app_web_log.socar_app_web_log` WHERE page_name='upsell_bottomsheet' AND event_date_kst >= '2026-06-10' AND REGEXP_CONTAINS(form_data, r'cu') GROUP BY step -- 결과: 노출 688, 옵션선택 251, CTA 670
2. 결제 수락, 매출 (서버 결제)
-- paid_meta_info(meta=CAR_UPGRADE)와 paid_info를 id로 조인 WITH cu_paid AS ( SELECT pi.reservation_id, pi.amount, DATE(pmi.created_at,"Asia/Seoul") AS dt FROM `socar-data.tianjin_replica.paid_meta_info` pmi JOIN `socar-data.tianjin_replica.paid_info` pi ON pi.id = pmi.id WHERE REGEXP_CONTAINS(pmi.meta, r'CAR_UPGRADE') AND DATE(pmi.created_at,"Asia/Seoul") >= '2026-06-10' ) SELECT dt, COUNT(DISTINCT reservation_id) AS accept, COUNT(DISTINCT reservation_id)*9900 AS customer_rev, SUM(amount) AS company_discount FROM cu_paid GROUP BY dt ORDER BY dt -- 결과: 6/10~6/15 수락 65예약, 고객매출 643,500, 회사부담 1,357,390 (환불 0)
3. 가드레일 A/B 예약 취소율 (버킷팅 + 헤비 계정 제외)
-- 버킷: farmhash(member_id + 'UPSELLING_CAR_UPGRADE') % 100 이 50 미만이면 노출군(B) -- (서버 코드 socar-common Testing.kt 의 ABTesting 공식. 노출자 688명 전원 B로 검증됨) -- 법인/대량 계정(6일 20건 이상)은 A쪽에 쏠려 예약수를 왜곡하므로 제외 WITH heavy AS ( SELECT member_id FROM `socar-data.tianjin_replica.reservation_info` WHERE DATE(created_at,"Asia/Seoul") BETWEEN '2026-06-10' AND '2026-06-15' AND member_id > 0 GROUP BY member_id HAVING COUNT(*) >= 20 ) SELECT MOD(ABS(FARM_FINGERPRINT(CAST(member_id AS STRING)||'UPSELLING_CAR_UPGRADE')),100) < 50 AS is_b, COUNT(*) AS rsv, COUNT(DISTINCT member_id) AS members, ROUND(COUNTIF(cancel_at IS NOT NULL)/COUNT(*)*100, 2) AS cancel_rate FROM `socar-data.tianjin_replica.reservation_info` WHERE DATE(created_at,"Asia/Seoul") BETWEEN '2026-06-10' AND '2026-06-15' AND member_id > 0 AND member_id NOT IN (SELECT member_id FROM heavy) AND zone_id = end_zone_id AND car_class_id IN (차종UP 대상 차급 id 목록) GROUP BY is_b -- 결과: A(대조) 33,212예약 26.84% vs B(노출) 33,453예약 26.83% (취소율 거의 동일 = 노출이 예약 방해 안 함)

용어 정리 및 유의사항

노출 (view)
차종UP 바텀시트를 본 B군 회원 수. form_data에 co/cu 키가 있으면 차종UP 노출
옵션 선택
상위 차급 카드를 실제로 고른 사람(car_upgrade_option click). 원래 차종 유지는 로그가 남지 않음
결제 수락
차종UP을 결제 완료한 예약. 결제 meta upselling_type CAR_UPGRADE, 예약 단위 합계 양수만 집계(환불 제외)
수락률 단위
수락률은 노출된 회원 중 수락 회원(58명 / 688명 = 8.4%). 시간연장과 동일 기준이라 직접 비교 가능. 전체 수락 59명 중 58명이 노출 로그 확인됨(1명 웹 이벤트 누락). 건수와 매출은 예약 단위(65예약)
고객 추가 매출
고객이 더 낸 금액. 9,900원 고정 x 수락 예약 수
회사 할인 부담 (참고)
상위 차급 차액 중 회사가 부담한 금액(paid_info의 upselling_discount 결제행 amount). 공헌이익 관점에서 상위 차종도 유휴였다면 손해로 보기 어려워 전면 지표로 두지 않음
예약 전환 가드레일
노출이 결제를 방해하는지(B 노출군 vs A 대조군 예약 전환율)는 모수가 더 쌓인 뒤 산출 예정. 시간연장도 표본 부족으로 유의성은 미확인이었음
CTA vs 옵션 선택
cta_button은 원래 차종으로 결제해도 찍히므로 수락 지표로 쓰지 않음. 실제 수락은 결제 meta로 판정
