실험 성패를 가르는 두 축이 통계적으로 확인됨
1) 수요 있음 (수락률이 우연이 아님) 2) 부작용 없음 (기존 예약 안 깨짐)
단 "유휴 상위 차로 회사가 순이익을 봤는지(그 차가 어차피 나갔을지)"는 현재 모수가 부족해 아직 못 봄. 노출 조건 넓혀 모수 키우면 대조군 대비로 측정 가능 (아래 한계)
수요 (통계적으로 유의)   노출 회원의 8.6%(59/688)가 차종UP 결제. z=4.3, 95% 신뢰구간 6.5~10.7%로 우연이 아님. 9,900원에 차종까지 바꿔야 하는데도 제안이 고객을 상위 차종으로 움직인다는 것이 확인됨
부작용 없음   노출군과 대조군의 예약 취소율 차이가 없음(27.12% vs 27.27%, z=0.44, 표본 각 3.3만 건). 차종UP을 띄워도 기존 예약을 깨지 않음 (아래 가드레일)
아직 못 보는 것 (회사 진짜 순이익)   지금 보이는 643,500원은 "고객이 더 낸 돈"임. 회사 입장의 진짜 순이익은 "그 상위 차종이 차종UP 없었어도 어차피 나갔을지"까지 빼야 나옴. 이건 차종UP을 켠 쪽(B)과 끈 쪽(A)의 전체 매출을 비교하면 측정됨. ABT가 회원을 무작위로 반반 나눠놔서 A군이 "제안 안 했을 때"를 그대로 대표하기 때문
왜 아직 안 보이나
실제로 6일치 A vs B 전체 매출을 재보면 A 평균 59,328원 vs B 58,949원으로, 차이가 오히려 -379원(B가 살짝 낮음). 차종UP이 만든 추가매출(64만원)이 비교 대상 전체 매출(약 10.9억)의 0.06%밖에 안 돼서, 예약 한두 건 차이로 생기는 일상적 매출 출렁임에 완전히 묻혀버림.
비유하면 수영장에 물 한 컵 붓고 수위가 올랐는지 재는 격. 효과가 없어서가 아니라, 측정 단위가 너무 커서 안 보이는 것.

어떻게 하면 보이나
1) 비교 범위를 "차종UP 노출되는 사람"으로 좁히면 신호가 진해짐(1인당 935원). 이 좁힌 비교에서 B가 A보다 확실히 높다고 말하려면 통계적으로 그룹당 약 4.9만 명이 필요(실측 객단가 분산 기준).
2) 현재 노출이 하루 약 115명이라 4.9만 명을 모으는 데 약 14개월. 노출 조건(대상 차급, 존, 리드타임)을 넓혀 하루 노출 수를 늘리면 그만큼 빨리 도달함.
결론: "측정 불가"가 아니라 "표본이 더 쌓이면 보이는" 문제. 노출을 키우는 게 순이익을 확인하는 길 (2차 시간연장도 같은 이유로 모수가 부족했음).

검증용 실제 쿼리 BigQuery
1. 노출, 옵션 선택, CTA 퍼널 (웹 이벤트)
-- 차종UP은 form_data에 co(원본 차종), cu(제안 차종) 키가 있어 cu 존재로 식별 SELECT CASE WHEN event_name='view' THEN '1_노출' WHEN component_name='car_upgrade_option' THEN '2_옵션선택' WHEN component_name='cta_button' THEN '3_CTA' END AS step, COUNT(DISTINCT member_id) AS uu FROM `socar-data.app_web_log.socar_app_web_log` WHERE page_name='upsell_bottomsheet' AND event_date_kst >= '2026-06-10' AND REGEXP_CONTAINS(form_data, r'cu') GROUP BY step -- 결과: 노출 688, 옵션선택 251, CTA 670
2. 결제 수락, 매출 (서버 결제)
-- paid_meta_info(meta=CAR_UPGRADE)와 paid_info를 id로 조인 WITH cu_paid AS ( SELECT pi.reservation_id, pi.amount, DATE(pmi.created_at,"Asia/Seoul") AS dt FROM `socar-data.tianjin_replica.paid_meta_info` pmi JOIN `socar-data.tianjin_replica.paid_info` pi ON pi.id = pmi.id WHERE REGEXP_CONTAINS(pmi.meta, r'CAR_UPGRADE') AND DATE(pmi.created_at,"Asia/Seoul") >= '2026-06-10' ) SELECT dt, COUNT(DISTINCT reservation_id) AS accept, COUNT(DISTINCT reservation_id)*9900 AS customer_rev, SUM(amount) AS company_discount FROM cu_paid GROUP BY dt ORDER BY dt -- 결과: 6/10~6/15 결제 65건, 고객매출 643,500, 회사부담 1,357,390 (환불 0)
3. 가드레일 A/B 예약 취소율 (버킷팅 + 헤비 계정 제외)
-- 버킷: farmhash(member_id + 'UPSELLING_CAR_UPGRADE') % 100 이 50 미만이면 노출군(B) -- (서버 코드 socar-common Testing.kt 의 ABTesting 공식. 노출자 688명 전원 B로 검증됨) -- 법인/대량 계정(6일 20건 이상)은 A쪽에 쏠려 예약수를 왜곡하므로 제외 -- 대상 차급 = 차종UP 노출에서 실제 원본(co)으로 제안된 차급을 로그에서 직접 추출(하드코딩 없이 재현) WITH co_list AS ( SELECT DISTINCT SAFE_CAST(REGEXP_EXTRACT(form_data, r'co\\?:\\?([0-9]+)') AS INT64) AS cid FROM `socar-data.app_web_log.socar_app_web_log` WHERE event_date_kst BETWEEN '2026-06-10' AND '2026-06-15' AND page_name='upsell_bottomsheet' AND event_name='view' AND REGEXP_CONTAINS(form_data, r'cu') ), heavy AS ( SELECT member_id FROM `socar-data.tianjin_replica.reservation_info` WHERE DATE(created_at,"Asia/Seoul") BETWEEN '2026-06-10' AND '2026-06-15' AND member_id > 0 GROUP BY member_id HAVING COUNT(*) >= 20 ) SELECT MOD(ABS(FARM_FINGERPRINT(CAST(member_id AS STRING)||'UPSELLING_CAR_UPGRADE')),100) < 50 AS is_b, COUNT(*) AS rsv, COUNT(DISTINCT member_id) AS members, ROUND(COUNTIF(cancel_at IS NOT NULL)/COUNT(*)*100, 2) AS cancel_rate FROM `socar-data.tianjin_replica.reservation_info` WHERE DATE(created_at,"Asia/Seoul") BETWEEN '2026-06-10' AND '2026-06-15' AND member_id > 0 AND member_id NOT IN (SELECT member_id FROM heavy) AND zone_id = end_zone_id AND car_class_id IN (SELECT cid FROM co_list WHERE cid IS NOT NULL) GROUP BY is_b -- 결과: A(대조) 33,681예약 27.12% vs B(노출) 33,940예약 27.27% (취소율 거의 동일, z=0.44 = 노출이 예약 방해 안 함)
4. 가입 보험(면책 자기부담금)별 차종UP 수락률
-- form_data pf = 진입 시점 적용 중인 면책상품(서버 protection_fee_info.deductible_type, 자기부담금 한도) -- form_data가 비이스케이프({pf:30..})와 이스케이프({\pf\:\30..}) 두 형태라 둘 다 잡는 정규식 사용 WITH exp AS ( SELECT member_id, REGEXP_EXTRACT(form_data, r'pf\\?:\\?([0-9A-Za-z_]+)') AS pf FROM `socar-data.app_web_log.socar_app_web_log` WHERE event_date_kst BETWEEN '2026-06-10' AND '2026-06-15' AND page_name='upsell_bottomsheet' AND event_name='view' AND REGEXP_CONTAINS(form_data, r'cu') AND member_id > 0 ), exp1 AS (SELECT member_id, ANY_VALUE(pf) AS pf FROM exp GROUP BY member_id), acc AS ( SELECT DISTINCT ri.member_id FROM `socar-data.tianjin_replica.paid_meta_info` pmi JOIN `socar-data.tianjin_replica.paid_info` pi ON pi.id = pmi.id JOIN `socar-data.tianjin_replica.reservation_info` ri ON ri.id = pi.reservation_id WHERE REGEXP_CONTAINS(pmi.meta, r'CAR_UPGRADE') AND DATE(pmi.created_at,'Asia/Seoul') BETWEEN '2026-06-10' AND '2026-06-15' ) SELECT e.pf, COUNT(DISTINCT e.member_id) AS exposed, COUNT(DISTINCT IF(a.member_id IS NOT NULL, e.member_id, NULL)) AS accepted FROM exp1 e LEFT JOIN acc a USING(member_id) GROUP BY e.pf ORDER BY exposed DESC -- 결과: 0_TOW(자기부담0원) 239명/23 9.6%, 30만원 208/21 10.1%, 70만원 221/10 4.5%, 5만원 20/4 20%(표본적음)

용어 정리 및 유의사항

노출 (view)
차종UP 바텀시트를 본 B군 회원 수. form_data에 co/cu 키가 있으면 차종UP 노출
옵션 선택
상위 차급 카드를 실제로 고른 사람(car_upgrade_option click). 원래 차종 유지는 로그가 남지 않음
결제 수락
차종UP을 결제 완료한 예약. 결제 meta upselling_type CAR_UPGRADE, 예약 단위 합계 양수만 집계(환불 제외)
수락률 단위
수락률은 수락 회원 59명 / 노출 회원 688명 = 8.6%. 결제 건수와 매출은 건 단위(65건). 한 회원이 여러 번 결제할 수 있어 사람(59)과 건수(65)를 구분
고객 추가 매출
고객이 더 낸 금액. 9,900원 고정 x 수락 예약 수
회사 할인 부담 (참고)
상위 차급 차액 중 회사가 부담한 금액(paid_info의 upselling_discount 결제행 amount). 공헌이익 관점에서 상위 차종도 유휴였다면 손해로 보기 어려워 전면 지표로 두지 않음
예약 전환 가드레일
노출이 결제를 방해하는지(B 노출군 vs A 대조군 예약 전환율)는 모수가 더 쌓인 뒤 산출 예정. 시간연장도 표본 부족으로 유의성은 미확인이었음
CTA vs 옵션 선택
