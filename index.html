기존 예약 방해 없음   차종UP 본 쪽(B)과 안 본 쪽(A)의 예약 취소율 거의 동일(26.98% vs 27.05%). 시트를 띄워도 원래 예약이 깨지지 않음 (아래 가드레일)
아직 못 보는 것: 회사 진짜 이득   지금 보이는 712,800원은 "고객이 더 낸 돈". 회사 진짜 이득은 "그 상위 차가 차종UP 없었어도 어차피 나갔을지"까지 빼야 나옴. 차종UP 켠 쪽(B)과 끈 쪽(A)의 전체 매출을 비교하면 확인 가능. 실험이 회원을 무작위로 반반 나눠놔서, A가 "제안 안 했을 때"를 그대로 보여주기 때문
왜 아직 안 보이나
7일치로 B(켠 쪽)와 A(끈 쪽)의 전체 매출을 비교하면 차이가 평균 207원(사실상 차이 없음). 차종UP이 만든 추가 매출(71만원)이 비교 대상 전체 매출(약 13.7억)의 0.05%라, 예약 한두 건 차이로 생기는 평소 매출 출렁임에 묻힘.
수영장에 물 한 컵 붓고 수위가 올랐나 재는 격. 효과가 없어서가 아니라 보는 범위가 너무 커서 안 보이는 것.

어떻게 하면 보이나
1) 비교 대상을 "차종UP 본 사람"만으로 좁히면 효과가 또렷해짐(한 명당 약 897원). 이 비교에서 B가 A보다 확실히 높다고 하려면 한 그룹당 약 5.9만 명 필요.
2) 지금은 하루 약 114명 노출이라 5.9만 명 모으려면 약 17개월. 노출 조건(대상 차급, 지역, 예약 시점) 넓혀 하루 노출 늘리면 그만큼 단축.
영영 못 보는 게 아니라 데이터가 더 쌓이면 보이는 문제. 노출을 늘리는 게 회사 이득을 확인하는 길 (시간연장 실험도 같은 이유로 데이터 부족).

검증용 실제 쿼리 BigQuery
1. 노출, 옵션 선택, CTA 퍼널 (웹 이벤트)
-- 차종UP은 form_data에 co(원본 차종), cu(제안 차종) 키가 있어 cu 존재로 식별 SELECT CASE WHEN event_name='view' THEN '1_노출' WHEN component_name='car_upgrade_option' THEN '2_옵션선택' WHEN component_name='cta_button' THEN '3_CTA' END AS step, COUNT(DISTINCT member_id) AS uu FROM `socar-data.app_web_log.socar_app_web_log` WHERE page_name='upsell_bottomsheet' AND event_date_kst BETWEEN '2026-06-10' AND '2026-06-16' AND REGEXP_CONTAINS(form_data, r'cu') GROUP BY step -- 결과: 노출 795, 옵션선택 279, CTA 775 (전체 기간 distinct. 일자별 distinct 합과는 다름)
2. 결제 수락, 매출 (서버 결제)
-- paid_meta_info(meta=CAR_UPGRADE)와 paid_info를 id로 조인 WITH cu_paid AS ( SELECT pi.reservation_id, pi.amount, DATE(pmi.created_at,"Asia/Seoul") AS dt FROM `socar-data.tianjin_replica.paid_meta_info` pmi JOIN `socar-data.tianjin_replica.paid_info` pi ON pi.id = pmi.id WHERE REGEXP_CONTAINS(pmi.meta, r'CAR_UPGRADE') AND DATE(pmi.created_at,"Asia/Seoul") BETWEEN '2026-06-10' AND '2026-06-16' ) SELECT dt, COUNT(DISTINCT reservation_id) AS accept, COUNT(DISTINCT reservation_id)*9900 AS customer_rev, SUM(amount) AS company_discount FROM cu_paid GROUP BY dt ORDER BY dt -- 결과: 6/10~6/16 결제 72건, 고객매출 712,800, 회사부담 1,424,240 (환불 0)
3. 가드레일 A/B 예약 취소율 (버킷팅 + 헤비 계정 제외)
-- 버킷: farmhash(member_id + 'UPSELLING_CAR_UPGRADE') % 100 이 50 미만이면 노출군(B) -- (서버 코드 socar-common Testing.kt 의 ABTesting 공식. 노출자 795명 전원 B로 검증됨) -- 법인/대량 계정(7일 20건 이상)은 A쪽에 쏠려 예약수를 왜곡하므로 제외 -- 대상 차급 = 차종UP 노출에서 실제 원본(co)으로 제안된 차급을 로그에서 직접 추출(하드코딩 없이 재현) WITH co_list AS ( SELECT DISTINCT SAFE_CAST(REGEXP_EXTRACT(form_data, r'co\\?:\\?([0-9]+)') AS INT64) AS cid FROM `socar-data.app_web_log.socar_app_web_log` WHERE event_date_kst BETWEEN '2026-06-10' AND '2026-06-16' AND page_name='upsell_bottomsheet' AND event_name='view' AND REGEXP_CONTAINS(form_data, r'cu') ), heavy AS ( SELECT member_id FROM `socar-data.tianjin_replica.reservation_info` WHERE DATE(created_at,"Asia/Seoul") BETWEEN '2026-06-10' AND '2026-06-16' AND member_id > 0 GROUP BY member_id HAVING COUNT(*) >= 20 ) SELECT MOD(ABS(FARM_FINGERPRINT(CAST(member_id AS STRING)||'UPSELLING_CAR_UPGRADE')),100) < 50 AS is_b, COUNT(*) AS rsv, COUNT(DISTINCT member_id) AS members, ROUND(COUNTIF(cancel_at IS NOT NULL)/COUNT(*)*100, 2) AS cancel_rate FROM `socar-data.tianjin_replica.reservation_info` WHERE DATE(created_at,"Asia/Seoul") BETWEEN '2026-06-10' AND '2026-06-16' AND member_id > 0 AND member_id NOT IN (SELECT member_id FROM heavy) AND zone_id = end_zone_id AND car_class_id IN (SELECT cid FROM co_list WHERE cid IS NOT NULL) GROUP BY is_b -- 결과: A(대조) 40,090예약 26.98% vs B(노출) 40,178예약 27.05% (취소율 거의 동일, z=0.22 = 노출이 예약 방해 안 함)
4. 가입 보험(면책 자기부담금)별 차종UP 수락률
-- form_data pf = 진입 시점 적용 중인 면책상품(서버 protection_fee_info.deductible_type, 자기부담금 한도) -- form_data가 비이스케이프({pf:30..})와 이스케이프({\pf\:\30..}) 두 형태라 둘 다 잡는 정규식 사용 WITH exp AS ( SELECT member_id, REGEXP_EXTRACT(form_data, r'pf\\?:\\?([0-9A-Za-z_]+)') AS pf FROM `socar-data.app_web_log.socar_app_web_log` WHERE event_date_kst BETWEEN '2026-06-10' AND '2026-06-16' AND page_name='upsell_bottomsheet' AND event_name='view' AND REGEXP_CONTAINS(form_data, r'cu') AND member_id > 0 ), exp1 AS (SELECT member_id, ANY_VALUE(pf) AS pf FROM exp GROUP BY member_id), acc AS ( SELECT DISTINCT ri.member_id FROM `socar-data.tianjin_replica.paid_meta_info` pmi JOIN `socar-data.tianjin_replica.paid_info` pi ON pi.id = pmi.id JOIN `socar-data.tianjin_replica.reservation_info` ri ON ri.id = pi.reservation_id WHERE REGEXP_CONTAINS(pmi.meta, r'CAR_UPGRADE') AND DATE(pmi.created_at,'Asia/Seoul') BETWEEN '2026-06-10' AND '2026-06-16' ) SELECT e.pf, COUNT(DISTINCT e.member_id) AS exposed, COUNT(DISTINCT IF(a.member_id IS NOT NULL, e.member_id, NULL)) AS accepted FROM exp1 e LEFT JOIN acc a USING(member_id) GROUP BY e.pf ORDER BY exposed DESC -- 결과: 0_TOW(자기부담0원) 292명/27 9.2%, 30만원 228/21 9.2%, 70만원 252/13 5.2%, 5만원 23/4 17.4%(표본적음)

용어 정리 및 유의사항

노출 (view)
차종UP 바텀시트를 본 B군 회원 수. form_data에 co/cu 키가 있으면 차종UP 노출
옵션 선택
상위 차급 카드를 실제로 고른 사람(car_upgrade_option click). 원래 차종 유지는 로그가 남지 않음
결제 수락
차종UP을 결제 완료한 예약. 결제 meta upselling_type CAR_UPGRADE, 예약 단위 합계 양수만 집계(환불 제외)
수락률 단위
수락률은 수락 회원 66명 / 노출 회원 795명 = 8.3%. 결제 건수와 매출은 건 단위(72건). 한 회원이 여러 번 결제할 수 있어 사람(66)과 건수(72)를 구분
고객 추가 매출
고객이 더 낸 금액. 9,900원 고정 x 수락 예약 수
회사 할인 부담 (참고)
상위 차급 차액 중 회사가 부담한 금액(paid_info의 upselling_discount 결제행 amount). 공헌이익 관점에서 상위 차종도 유휴였다면 손해로 보기 어려워 전면 지표로 두지 않음
예약 전환 가드레일
